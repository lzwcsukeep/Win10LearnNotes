# static_cast

- static_cast做了什么？什么时候用？

![](..\img\2024-05-16-11-52-25-image.png)

- `static_cast`在继承体系下面向下或者向上转型的时候会在子类对象内部移动指针。使得指针指向对应的父类的位置。

![](..\img\2024-05-16-11-48-37-image.png)

- `static_cast` 在向下转型的时候，如果是不同子类之间转换，并不会给出错误提示。(之前你自己试验过了的)

![](..\img\2024-05-16-12-02-36-image.png)

- static_cast ： 进行隐式的类型转换，以及自定义的类型转换。可以用于基本数据类型之间。

- static_cast 也可以用在父子类之间类型转换，但是不会检查，在编译期就完成转换。

# dynamic_cast

用途仅限于继承体系下面，有多态的情况下。父类和子类之间进行强制类型转换。这个过程是在runtime 完成的，并且会check。check不通过返回null。

# reinterpret_cast

最不安全的一种强制类型转换，直接改变底层 bit 位的解释方法。

### static_cast 会改变指针的地址值。

### 之所以要有这四种不同的强制类型转换是为了区分不同的意图。提高代码的可读性以及降低强制类型转换的危险性
