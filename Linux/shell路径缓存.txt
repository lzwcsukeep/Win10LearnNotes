**什么是 Shell 路径缓存？
当你在命令行输入一个命令（比如 svn）时，shell（比如 bash）会在环境变量 PATH 指定的目录列表中依次查找该命令的可执行文件位置。

为了提高效率，shell 会“缓存”这个查找结果，也就是说：

当你第一次运行 svn，shell 会搜索 PATH 里的目录，找到 /usr/bin/svn（举例），然后把这个路径记住。

以后你再运行 svn，shell 不会再重新搜索 PATH，直接调用之前缓存的路径。

**为什么会出现缓存问题？
当你：

安装了新版本的命令（比如 /usr/local/bin/svn），

或者删除了旧版本（比如 /usr/bin/svn），

但 shell 之前缓存的路径仍指向旧文件，就会出现 “找不到文件” 的错误。

如何清除这个缓存？
使用命令：
hash -r
hash -r 会清除 shell 里所有已缓存的命令路径，下一次运行命令时会重新搜索 PATH。

常见场景：换版本、删除或移动可执行文件后，执行 hash -r 让 shell 更新路径缓存。
