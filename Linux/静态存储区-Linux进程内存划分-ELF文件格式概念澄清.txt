1. 静态存储区是编译器/语言层面的概念
静态存储区是为了区分 栈内存和堆内存引入的概念。在 C/C++ 里，我们说的 静态存储区，就是用来存放：
全局变量、静态变量（static 关键字修饰的局部/全局变量）、常量数据（如字符串常量），
这些对象在程序整个运行周期中都存在，所以叫“静态存储期”。
静态存储区的概念和运行时内存划分，和ELF 都无关。

2. Linux 进程运行时内存划分

低地址
 ┌───────────────────────────────┐
 │   代码段 (Text Segment)                                        │  ← ELF 的 .text、部分 .rodata
 │   [只读 + 可执行]                                                     │
 ├───────────────────────────────┤
 │   数据段 (Data Segment)                                       │  ← ELF 的 .data（已初始化全局/静态变量）
 │   [可读 + 可写]                                                        │
 ├───────────────────────────────┤
 │   BSS 段 (BSS Segment)                                        │  ← ELF 的 .bss（未初始化全局/静态变量）
 │   [可读 + 可写，运行时清零]                                  │
 ├───────────────────────────────┤
 │   堆 (Heap)                                                             │  ← 动态分配：malloc/new
 │   [向高地址增长]                                                     │
 ├───────────────────────────────┤
 │   ... 共享库映射区 ...                                               │  ← ld.so 加载的 .so 文件
 │   [代码段/数据段混合]                                            │
 ├───────────────────────────────┤
 │   栈 (Stack)                                                             │  ← 局部变量、函数调用栈帧
 │   [向低地址增长]                                                     │
 ├───────────────────────────────┤
 │   内核空间 (Kernel space)                                      │  ← 用户态不可访问
 └───────────────────────────────┘
高地址

注： 进程运行时内存划分有代码段，数据段，BSS段。你之前只记得栈内存，堆，.so,内核空间。

3. ELF 文件格式划分

----
section 举例有 .text .rodata .bss .data。

section 是ELF 文件内容的基本单位。
segment 由多个section 组成，是加载器加载部分文件内容进内存的参考数据结构。

注：ELF 文件划分是文件的格式划分，是处在磁盘上的。和内存划分没有关系，只是这个文件会按segment
加载进内存。这个时候有一些对应关系， 比如 .text 会加载进代码段。

总结： 注意区分三个层次的关系： 

1. 编译器语言层面-- 静态存储区
2. 进程运行时内存划分 -- 代码段、数据段、BSS段
3. ELF 文件格式的section 划分 -- .text .rodata .bss .data -- 这些都是elf 文件的section. 文件是放在磁盘上的
